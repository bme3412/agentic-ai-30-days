Flow Code

class AnalysisFlow(Flow):
    @start()
    def gather_data(self):
        return {"data": crew.kickoff(self.inputs).raw}

    @listen(gather_data)
    def analyze_data(self, state):
        analysis = analyze(state["data"])
        return {"confidence": analysis.confidence}

    @router(analyze_data)
    def route_by_confidence(self, state):
        if state["confidence"] > 0.8:
            return "approve"
        return "review"

    @listen("approve")
    def auto_approve(self, state):
        return {"status": "approved", "process": "auto"}

    @listen("review")
    def human_review(self, state):
        return {"status": "pending_review", "process": "manual"}

Key Insights

  • Task controls let you add checkpoints, validation, and parallelism to your workflows
  • Tool descriptions are prompts in disguise—clear descriptions lead to better tool usage
  • Flows enable complex orchestration that sequential/hierarchical processes can't handle
  • State accumulates through a Flow—each method can read from and add to shared state
  • @router returns a string that matches a @listen decorator for conditional branching
