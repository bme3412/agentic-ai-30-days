Guardrails: Input & Output Validation

Guardrails validate agent inputs and outputs. Input guardrails block dangerous requests before they reach the LLM. Output guardrails filter sensitive data from responses.

Example Guardrail Code

async def block_prompt_injection(input_text: str) -> GuardrailResult:
    """Detect and block prompt injection attempts."""
    patterns = ["ignore previous", "system:", "you are now"]
    for pattern in patterns:
        if pattern.lower() in input_text.lower():
            return GuardrailResult(passed=False, message="Blocked.")
    return GuardrailResult(passed=True)

agent = Agent(
    name="SecureBot",
    input_guardrails=[InputGuardrail(func=block_prompt_injection)]
)

Key Insights

  • Four primitives—Agents, Tools, Handoffs, Guardrails. Learn these and you know the framework.
  • Tools are plain functions—type hints become schemas, docstrings become descriptions.
  • Handoffs preserve context—the new agent receives full conversation history.
  • Guardrails run before/after—input guardrails block bad requests, output guardrails filter responses.
  • Runs on your infrastructure—you control data, costs, and can use any Chat Completions-compatible LLM.
