Type-Parameterized Agents

Every PydanticAI agent is generic over two types: DepsT (dependencies) and OutputT (result type). Your IDE and type checker enforce these contracts.

Configure Agent Types

Agent Definition
Agent[AppDeps, WeatherReport](
    'openai:gpt-4o',
    deps_type=AppDeps,
    output_type=WeatherReport
)
↓
Run Result
result: AgentRunResult[WeatherReport]

# Type-safe access:
result.output.temperature  # int
result.output.condition    # str

Tools with Automatic Schema Generation

Decorate functions with @agent.tool and PydanticAI auto-generates JSON schemas from type hints. Docstrings become parameter descriptions.

Define a Tool

Generated JSON Schema

{
  "name": "search_products",
  "description": "Search the product catalog.",
  "parameters": {
    "type": "object",
    "properties": {
      "query": {
        "type": "string",
        "description": "Search terms to find products"
      },
      "category": {
        "type": "string",
        "description": "Product category filter",
        "default": "all"
      },
      "limit": {
        "type": "integer",
        "description": "Maximum results to return",
        "default": 10
      }
    },
    "required": ["query"]
  }
}
💡 No decorators needed - type hints are the schema
💡 Docstrings parsed - Google, NumPy, or Sphinx format

Dependency Injection

Follow the FastAPI pattern: define dependencies as dataclasses, pass the type to the agent, and access via RunContext. Override for testing without changing agent code.

1

Define Dependencies

@dataclass
class AppDeps:
    http_client: httpx.AsyncClient
    api_key: str
    user_id: str
→
2

Inject via Agent

agent = Agent(
    'openai:gpt-4o',
    deps_type=AppDeps
)
→
3

Access in Tools

@agent.tool
async def fetch(ctx: RunContext[AppDeps]):
    return await ctx.deps.http_client.get(
        headers={"Auth": ctx.deps.api_key}
    )

Testing with Override

Production
deps = AppDeps(client=RealClient(), api_key="sk-xxx")
Testing
with agent.override(deps=MockDeps()):
    result = agent.run_sync("test")
✔ Same agent code, different dependencies. No mocking frameworks needed.

Output Validation with Automatic Retry

When you specify an output_type, PydanticAI validates LLM responses against the schema. Failed validation triggers retry with the error message, letting the model self-correct.

Pydantic Model

class MovieReview(BaseModel):
    title: str
    rating: int = Field(ge=1, le=10)
    summary: str = Field(max_length=200)
    recommend: bool
Select an output scenario and click "Run Validation"

How Retry Works

1 LLM generates structured output
2 Pydantic validates against schema
3 If invalid, error feeds back to LLM
4 LLM corrects and retries (up to retries=N)

Key Insights

  • Type hints are the contract—DepsT and OutputT are enforced by IDE and type checker.
  • Tools auto-generate schemas—no decorators or manual registration needed.
  • Dependencies are injectable—override() swaps them for testing without code changes.
  • Validation triggers retry—errors feed back to the LLM for self-correction.
  • Model agnostic—50+ providers: OpenAI, Anthropic, Google, Groq, and more.
