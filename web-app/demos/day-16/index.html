Checkpointing Enables

  • Pause/Resume: Stop execution, restart later, continue exactly where you left off
  • Time Travel: Jump to any previous state for debugging or branching
  • Human-in-the-Loop: Interrupt before sensitive operations for approval
  • Fault Tolerance: Recover from crashes without losing progress

Choosing a Memory Type

  • BufferMemory: Complete history, best for short conversations (<10 turns)
  • SummaryMemory: Fixed tokens, good for long chats but loses details
  • WindowMemory: Simple and predictable, but forgets early context
  • SummaryBufferMemory: Best balance - summarizes old, keeps recent verbatim

Thread Manager

Thread IDs organize conversations. Each thread maintains isolated state, enabling multi-user systems where contexts don't leak between users.

Thread ID Patterns

  • User + Session: user-{'{user_id}'}-session-{'{session_id}'}
  • With Timestamp: user-{'{user_id}'}-{'{YYYYMMDD-HHMMSS}'}
  • Anonymous: anon-{'{uuid}'}
  • Isolation: Different thread IDs = completely separate conversations

Choosing a Checkpointer

  • MemorySaver: Fast, no setup, but loses everything on restart. Dev only.
  • SqliteSaver: File-based persistence, survives restarts. Single server.
  • PostgresSaver: Database persistence, multi-instance support. Production.
  • AsyncPostgresSaver: Same as above with async support. High traffic.

Key Insights

      
    
        
  • LLMs are stateless by default - all memory must be managed by your application
    • 
        
  • LangGraph checkpointing saves full graph state at every step, enabling pause/resume and time travel
    • 
        
  • Thread IDs organize conversations - same thread_id = continued conversation
    • 
        
  • Use MemorySaver for development, SqliteSaver for single-instance, PostgresSaver for production
    • 
        
  • Conversation memory (Buffer, Summary, Window, SummaryBuffer) manages token usage
    • 
        
  • SummaryBufferMemory offers the best balance of context retention and token efficiency
    • 
        
  • Production agents often combine checkpointing, conversation memory, AND vector stores
