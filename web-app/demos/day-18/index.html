Caching Strategies Comparison

Compare different caching approaches: exact match, semantic, and hybrid strategies.

Exact Match

Simple

Cache based on exact query string matching.

Pros
  • Zero false positives
  • Fast lookup (O(1))
  • Easy to implement
Cons
  • Low hit rate
  • Misses paraphrases
  • Case/punctuation sensitive
"What is AI?" = "What is AI?" HIT
"What is AI?" ≠ "what is ai?" MISS

Semantic Cache

Recommended

Cache based on meaning similarity using embeddings.

Pros
  • High hit rate (40-80%)
  • Handles paraphrases
  • Language-agnostic
Cons
  • Embedding compute cost
  • Tunable threshold needed
  • Possible false positives
"What is AI?" ~ "Explain artificial intelligence" HIT (0.94)
"What is AI?" ~ "How to code in Python?" MISS (0.31)

Hybrid Cache

Advanced

Combine exact match first, then semantic fallback.

Pros
  • Best of both worlds
  • Exact match is instant
  • Highest overall hit rate
Cons
  • More complex logic
  • Two storage systems
  • Higher memory usage

Redis Vector Search Commands

Learn the Redis Stack commands for building semantic caches with vector similarity search.

1. Create Vector Index

FT.CREATE cache_idx ON HASH PREFIX 1 cache:
SCHEMA
  query TEXT
  response TEXT
  embedding VECTOR HNSW 6
    TYPE FLOAT32
    DIM 1536
    DISTANCE_METRIC COSINE
HNSW: Fast approximate search algorithm
DIM 1536: OpenAI text-embedding-3-small dimension
COSINE: Best for normalized embeddings

2. Store Cache Entry

HSET cache:12345
  query "What is machine learning?"
  response "Machine learning is a subset of AI..."
  embedding 
  timestamp 1709856000
  model "gpt-4o"
Store query, response, and embedding vector together.
Add metadata like timestamp and model for cache management.

3. KNN Similarity Search

FT.SEARCH cache_idx
  "*=>[KNN 3 @embedding $vec AS score]"
  PARAMS 2 vec 
  RETURN 3 query response score
  SORTBY score
  LIMIT 0 1
  DIALECT 2
KNN 3: Find 3 nearest neighbors
score: Cosine distance (0 = identical, 2 = opposite)
DIALECT 2: Required for vector queries

4. Check Cache Hit (Threshold Filter)

# In Python, check if score meets threshold:
similarity = 1 - (score / 2)  # Convert distance to similarity
if similarity >= 0.92:
    return cached_response
else:
    call_llm_and_cache()
Redis returns cosine distance, not similarity.
Convert: similarity = 1 - (distance / 2)

5. Cache Expiration (TTL)

# Set TTL on cache entries
EXPIRE cache:12345 86400  # 24 hours

# Or use HSET with EXAT
HSET cache:12345 ... EXAT 1709942400
Use TTL for time-sensitive data or to manage cache size.
Common TTLs: 1 hour (3600s), 24 hours (86400s), 7 days (604800s)
