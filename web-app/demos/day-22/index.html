Why REST Instead of JSON-RPC?

ACP (REST)
# List agents
curl GET /agents

# Create run
curl POST /runs -d '{"agent": "researcher", "input": {...}}'

# Get status
curl GET /runs/abc123
  • ✅ Standard HTTP methods
  • ✅ Test with browser/Postman
  • ✅ Works with CDNs/proxies
  • ✅ No special libraries
MCP/A2A (JSON-RPC)
{
  "jsonrpc": "2.0",
  "method": "message/send",
  "params": { "message": {...} },
  "id": "req-123"
}
  • ⚠️ Requires parsing envelope
  • ⚠️ Needs dedicated SDK
  • ⚠️ Method in body, not URL
  • ⚠️ Less tooling support
