Reflection & Self-Improvement

Teaching agents to critique and improve their own outputs

What This Demo Teaches

Reflection patterns let AI agents improve their outputs through self-critique. Instead of accepting the first response, the model evaluates its work, identifies issues, and refines iteratively. This demo shows when and how self-improvement works.

📝

Generate

The model produces an initial output—code, text, or a solution to a problem.

🔍

Critique

A separate evaluation step identifies specific issues, often using external tools for grounding.

⟳

Refine

Based on the critique, the model revises its output to address identified problems.

Key insight: Self-correction works for surface errors (formatting, style) but fails for reasoning errors without external grounding. The critic needs new information.

Reflexion: Learn Across Attempts

Unlike Self-Refine, Reflexion stores verbal reflections in memory. Each failure teaches a lesson.

Attempt 1 ✗ Failed 
def two_sum(nums, target):
    for i in range(len(nums)):
        for j in range(len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
Test Failed: two_sum([3, 3], 6) returned [0, 0], expected [0, 1]
Reflection: "My inner loop starts at 0, allowing the same element to be used twice. For next time: start the inner loop at i+1 to ensure distinct indices."
↓ Reflection stored in memory
Attempt 2 ✓ Passed 
def two_sum(nums, target):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):  # Start after i
            if nums[i] + nums[j] == target:
                return [i, j]
All tests passed

Key Difference: External Feedback

Reflexion works because the environment (test results) provides the failure signal—not the model's self-assessment. The reflection is informed by concrete external feedback.
