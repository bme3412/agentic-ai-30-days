LangChain Tools & Agents

Building agents that can take action in the real world

What This Demo Teaches

Tools give agents the ability to interact with the world beyond text generation. Instead of just responding, agents can search the web, query databases, call APIs, and execute code. This demo shows how agents decide which tools to use and how the tool-calling flow works.

🔧

Define Tools

Create tools with clear descriptions so the model knows when to use them.

🧠

Bind & Reason

The model analyzes the query and decides which tool(s) to call with what arguments.

⚡

Execute & Loop

Tools execute, results feed back, and the agent continues until the task is complete.

Key insight: The quality of your tool descriptions directly impacts how reliably the agent will use them. Clear, specific descriptions > vague ones.

Available Tools

🔍 web_search

Search the web for current information. Use for recent events, facts, or when you need up-to-date data.

query: str - The search query
🧮 calculator

Evaluate mathematical expressions. Use for arithmetic, percentages, or any numerical calculations.

expression: str - Math expression (e.g., "15 * 7 + 3")
🌦 get_weather

Get current weather for a city. Use when users ask about weather, temperature, or conditions.

city: str - City name (e.g., "Tokyo")
📖 wikipedia

Search Wikipedia for factual information about people, places, events, or concepts.

topic: str - Topic to look up
🗃 query_database

Execute read-only SQL queries against the database. Use for data lookups and reports.

sql: str - SELECT query only

What's Happening

The agent analyzes the query, identifies what information it needs, and selects the appropriate tool(s). For complex queries, it may call multiple tools in sequence or parallel.

Tool Calling Flow

Step through the complete lifecycle of a tool call, from binding to execution.

1

bind_tools()

Attach tools to the model. The model learns what tools are available and their schemas.

model = ChatOpenAI(model="gpt-4")
model_with_tools = model.bind_tools([
    get_weather,
    calculator,
    web_search
])
↓
2

Model Invocation

Send the user's message. The model decides if tools are needed.

response = model_with_tools.invoke([
    HumanMessage("What's the weather in Tokyo?")
])
↓
3

Tool Calls Generated

The model outputs structured tool calls instead of (or in addition to) text.

response.tool_calls = [
    {
        "id": "call_abc123",
        "name": "get_weather",
        "args": {"city": "Tokyo"}
    }
]
↓
4

Execute Tools

Your code runs the actual tool functions with the provided arguments.

for tool_call in response.tool_calls:
    result = tools[tool_call["name"]].invoke(
        tool_call["args"]
    )
    # result = "Tokyo: 22°C, Partly cloudy"
↓
5

Feed Results Back

Package results as ToolMessages and continue the conversation.

messages = [
    HumanMessage("What's the weather in Tokyo?"),
    response,  # AIMessage with tool_calls
    ToolMessage(
        content="Tokyo: 22°C, Partly cloudy",
        tool_call_id="call_abc123"
    )
]
final = model_with_tools.invoke(messages)
↓
6

Final Response

Model synthesizes the tool results into a natural language response.

final.content = "The weather in Tokyo is currently
22°C (72°F) with partly cloudy skies."
