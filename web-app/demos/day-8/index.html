Persisting agent state across sessions

What This Demo Teaches

Checkpointing saves agent state after each step, enabling conversation memory, crash recovery, and human-in-the-loop workflows. This demo visualizes how state persists and how thread IDs isolate conversations.

Code Pattern

from langgraph.checkpoint.memory import MemorySaver
from langgraph.graph import StateGraph

# Enable checkpointing
checkpointer = MemorySaver()
agent = graph.compile(checkpointer=checkpointer)

# Each thread_id gets isolated state
config = {"configurable": {"thread_id": "user-123"}}

# Conversation persists across invocations
result1 = agent.invoke({"messages": [("human", "Hi, I'm Alice")]}, config)
result2 = agent.invoke({"messages": [("human", "What's my name?")]}, config)
# Agent remembers: "Your name is Alice"

# Human-in-the-loop: pause and resume
agent.update_state(config, {"approved": True})
result = agent.invoke(None, config)  # Resume from checkpoint
